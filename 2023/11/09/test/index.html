The directory strucure of Hexo

Going into the root directory of Hexo’s Blog you can see the following structure:

  • _config.yml is the configuration file for the entire blog site, which contains the site’s configuration, URL configuration, directory configuration, directory configuration, theme configuration, and more. It’s easy to understand and use.

  • scaffolds is the template fold, there are three template files in it, in order, these are draft templates, page templates and article templates.

    When we use the Hexo command to generate a draft, page or article, we are copying these templates. Let’s take the article template as an example for illustration.

The default post.md content is empty and the header information is as follows:

If we want all newly generated posts to carry the category field, we can modify the header of post.md. we add the categories field to the header.

This way when we use hexo new postName to generate a new article, this article will all have the categories field at the top.

  • resources folder is where user resources are stored. Files/folders starting with _(underscore) and hidden files are ignored, except for the _posts folder; Markdown and HTML files are parsed and palced in the public folder, while other files are copied there.
